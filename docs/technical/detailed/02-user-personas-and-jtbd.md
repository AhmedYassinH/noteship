# Noteship — User Personas & Jobs-to-Be-Done

## Purpose

Capture primary users, motivations, and critical workflows.

## Persona A: Solo Consultant (Primary)

**Context:** Works with multiple clients, repeats similar frameworks, writes follow-up notes and proposals.

### Pain points

- Can’t find old notes when needed (doesn’t remember wording)
- Spends time recreating assets (frameworks, checklists, templates)
- Wants steady publishing but struggles with execution

### JTBD

- _When I need to recall an idea from past work, help me find the right note fast even if I only remember the concept._
- _When I want to publish, help me turn raw notes into a good post and ship it with minimal friction._

### Success

- Finds relevant note in <10 seconds
- Publishes consistently (scheduled posts)
- Saves time weekly

## Persona B: Coach (Primary)

**Context:** Works with clients 1:1, has recurring principles, creates educational content to build authority.

### Pain points

- Notes scattered; can’t retrieve insights quickly
- Wants voice consistency and “my tone” in posts
- Needs a simple content calendar

### JTBD

- _When I prepare for a session, help me recall prior insights and actions quickly._
- _When I want to build my brand, help me repurpose coaching insights into posts._

## Secondary personas (later)

- Content creators (more volume, more analytics needs)
- Educators (imports from meeting tools; structured flashcards)

## Key workflows (MVP)

1. Create/edit note
2. Semantic search across notes
3. Generate post drafts from a note (tone/persona)
4. Publish now OR schedule to LinkedIn/Medium
5. See post status (draft/scheduled/published/failed)
