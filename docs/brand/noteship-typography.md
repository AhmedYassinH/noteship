# Noteship — Typography System (App + Marketing)

_Last updated: 2026-01-24_

## 0) Locked font strategy

### Dashboard / App UI (utility-first)

- **EN:** IBM Plex Sans
- **AR:** IBM Plex Sans Arabic

### Public-facing / Marketing pages (modern editorial)

- **Headlines (EN):** Lora
- **Headlines (AR):** Noto Naskh Arabic
- **Body/UI (EN):** IBM Plex Sans
- **Body/UI (AR):** IBM Plex Sans Arabic

## 1) Usage rules (non-negotiable)

- **Serif fonts are headlines only** on marketing pages.
- **No serif in buttons, nav, forms, pricing tables, or dashboard.**
- App/dashboard uses **IBM Plex only** (both languages).

## 2) Recommended weights

### IBM Plex Sans (EN) + IBM Plex Sans Arabic (AR)

- Body: 400
- UI emphasis: 500
- Section titles: 600

### Lora (EN headlines)

- H1/H2: 600 (or 700 if needed)
- H3: 600

### Noto Naskh Arabic (AR headlines)

- Headline: 600 (avoid extreme bold)
- Subhead: 500–600

## 3) Type scale (suggested)

Marketing:

- H1: 40–52
- H2: 28–36
- H3: 20–24
- Body: 16–18
- Small: 13–14

App:

- Page title: 24–28
- Section title: 16–18 (600)
- Body: 14–16
- Small: 12–13

## 4) Line-height guidelines

- English body: 1.5–1.7
- Arabic body: 1.7–1.9 (Arabic needs more breathing room)
- Headlines: 1.1–1.25 (both)

## 5) Spacing rhythm

- Use consistent vertical rhythm:
  - Paragraph spacing > line breaks
  - Keep Arabic paragraph spacing slightly larger if text is dense

## 6) Fallbacks

- EN fallback: system-ui, Segoe UI, Arial
- AR fallback: system Arabic fonts, Arial

## 7) Anti-patterns

- Mixing serif + sans within the same headline line
- Using Lora in dashboard
- Using Naskh for body paragraphs in dense UI tables (keep for editorial headlines only)
